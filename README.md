# Tableau Portfolio — AI Adoption ROI (Stack Overflow Developer Survey 2025)

**Live Tableau dashboard:** https://public.tableau.com/views/StackOverflowAnnualDeveloperSurveyVisualizingGlobalDeveloperTrendsin2025/HowAIAdoptionShapesDeveloperPaySatisfactionProductivity?:language=en-US&:sid=&:redirect=auth&:display_count=n&:origin=viz_share_link

This repo is built to showcase **Tableau dashboarding** *and* the **full data lifecycle** behind it (Python → engineered features → Tableau-ready extracts → story-driven dashboards).

---

## Dataset
- **53,853** raw responses
- **177** countries represented
- Survey covers developer roles, experience, tools, AI usage, satisfaction, compensation, and org enablement signals.

---

## Why this analysis matters
- Quantify the value of AI adoption (pay, satisfaction, workflow/tool complexity)
- Identify **who benefits most** (and where risk pockets show up)
- Understand what enables ROI (training, tool access, policies/compute)
- Translate patterns into **practical levers** leaders can use

---

## Research questions
1. **AI intensity → pay & productivity proxies** (ATE-style comparisons + cohort slices)
2. **Skill complementarity**: which foundations (SQL/cloud/language breadth) amplify AI impact
3. **Heterogeneity**: who gains (or loses) across role/experience/remote/org size/region
4. **Barriers to AI ROI**: enablement, policy, compute, ecosystem factors driving adoption
5. **Policy simulation**: which levers move adoption/outcomes most with manageable friction

---

## Key engineered variables (high level)
- **AI usage:** `AI_IntensityScore`, `HighAI` (top ~20% of AI intensity)
- **Outcomes:** `Pay_USD` (winsorized), `LogPay`, job satisfaction, tool usage proxies, friction/threat signals
- **Skills:** counts of languages / DBs / cloud tools, plus flags like SQL/BI usage
- **Controls / cohorts:** developer type, experience band, org size, remote status, region/country, education, etc.

---

## What stood out (summary insights)
- **AI adoption shows strong economic value**: higher pay, higher satisfaction, and higher productivity proxies across many cohorts.
- **Skill foundations multiply AI impact**: cloud + SQL + broader tool depth are associated with larger uplift.
- **Organizational support drives ROI**: training access, compute/tool availability, and culture/enablement correlate with higher adoption and better outcomes.

> Note: The survey is self‑reported and observational. Findings reflect **patterns and associations**, not strict causation.

---

## Tableau developer highlights
- Designed dashboards to compare **HighAI vs LowAI** across pay, satisfaction, and productivity proxies
- Built cohort slicing views (role, experience, remote work, org size, region) to surface heterogeneity
- Added “policy lever” views to prioritize enablement actions (training/tool access first; agents as advanced pilots)

## Repo structure
- `notebooks/` — reproducible Python notebooks (prep → modeling → Tableau-long tables)
- `data/sample/` — small samples + schema (safe to commit)
- `data/raw/` — **not committed** (place the full survey CSV here locally)
- `data/processed/` — Tableau-ready extract generated by Notebook 01
- `outputs/tables/` — compact tables used by Tableau levers/ROI views
- `dashboards/` — dashboard link + screenshots

---

## How to run (local)
1. Create environment and install deps:
   ```bash
   pip install -r requirements.txt
   ```
2. Place the full survey file locally:
   - `data/raw/survey_results_public.csv`
3. Run notebooks in order:
   - `01_data_prep_feature_engineering.ipynb` → creates `data/processed/so2025_tableau_clean_final.csv`
   - `02_modeling_policy_levers.ipynb` → creates policy lever impact tables
   - `03_convert_long_format.ipynb` → creates long-format tables for Tableau

---

## Limitations
- Self‑reported survey data (possible bias)
- Respondents skew technical
- Some multi-select tool/skill counts can be noisy
- Observational data → interpret as patterns, not strict causality
